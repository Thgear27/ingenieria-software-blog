La estimación de historias de usuario es un paso esencial para asegurar una planificación efectiva en el marco de trabajo Scrum. Permite al equipo de desarrollo identificar el esfuerzo necesario para cada funcionalidad, facilitando la organización y gestión del backlog.

### Historias de Usuario

- **Historia 1:** Como usuario, quiero registrarme fácilmente a través de un formulario intuitivo o mi cuenta de Google, para agilizar el proceso de creación de cuenta.
- **Historia 2:** Como usuario, quiero hacer una reserva de cita a través de un enlace directo al chat de Instagram de la empresa, para confirmar rápidamente con la dueña.
- **Historia 3:** Cómo dueña, quiero tener un visor de las citas pendientes y confirmadas, para organizar mis servicios de manera eficiente y asegurar una buena atención al cliente.


## HERRAMIENTAS

### Métodos de Estimación

Para estimar las historias de usuario, el equipo Scrum utilizará el método **Planning Poker**. Este método facilita la discusión y el consenso sobre el esfuerzo requerido para cada historia, permitiendo asignar puntos de manera colaborativa.
