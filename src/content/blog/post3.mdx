Desarrollar épicas en la metodología Scrum es crucial para LIENZO Estudio, ya que
permite tener una visión global del proyecto y descomponer las grandes funcionalidades en
historias de usuario más manejables. Esto facilita la priorización de las características que
aportarán mayor valor al negocio, asegurando que el equipo trabaje en lo más relevante primero.
Además, las épicas fomentan la colaboración entre las propietarias, los stakeholders y el grupo de Scrum,
alineando sus expectativas y objetivos.

# SALIDAS

## Épicas

Las épicas son esenciales porque proporcionan una visión general de las funcionalidades
del producto, ayudan a priorizar tareas y permiten descomponer el trabajo en historias de usuario
más manejables. Fomentan la colaboración entre el equipo y los stakeholders, ofrecen flexibilidad
ante cambios en los requisitos y facilitan el seguimiento del progreso, asegurando que el trabajo
esté alineado con los objetivos estratégicos de la organización y maximizando el impacto de las
entregas.

### Épica 1: Página de bienvenida

**Descripción:** El sistema debe presentar una página de bienvenida cuando se ingrese al sitio web, funcionando como 
una “landing page” atractiva visualmente, con fácil acceso a las principales secciones y manteniendo 
una carga rápida.

- **Criterios de éxito:**
  <ul style={{ listStyleType: "circle", paddingLeft: "20px" }}>
    <li>Diseño Visual Atractivo: La página debe contar con colores vibrantes, imágenes de alta calidad y videos integrados para captar la atención del usuario.</li>
    <li>Carga Ágil: A pesar del contenido visual, la página debe estar optimizada para cargar rápidamente en diferentes dispositivos y conexiones a internet, asegurando una buena experiencia de usuario.</li>
    <li>Facilidad de Navegación: La interfaz debe ser sencilla y amigable, utilizando distintas tipografías para destacar diferentes secciones, y permitiendo al usuario navegar de manera intuitiva sin miedo o frustración.</li>
    <li>Footer: Incluir columnas con el logo, horario de atención y enlaces a redes sociales, dando menor énfasis a la política de privacidad y el libro de reclamaciones, pero haciéndolos accesibles.</li>
    <li>
      Barra de Navegación: Debe incluir secciones clave como:
      <ul style={{ listStyleType: "square", paddingLeft: "20px" }}>
        <li>Promociones: Destacar ofertas y promociones activas.</li>
        <li>Servicios: Mostrar los servicios ofrecidos de manera clara y ordenada.</li>
        <li>Reservas: Facilitar el proceso de reservar una cita de forma rápida y sencilla.</li>
        <li>Ubícanos: Incluir un mapa enmarcado de Google Maps para mostrar la ubicación de las sedes.</li>
        <li>Galería: Mostrar una cuadrícula de imágenes (3x3) con contenido visual atractivo e integración con el Instagram del negocio.</li>
        <li>Logueate Fácil: Simplificar el proceso de login para que los usuarios no se sientan abrumados o pierdan interés en registrarse mediante vinculación de google con dos simples pasos.</li>
      </ul>
    </li>
  </ul>

### Épica 2: Gestión de Citas de Clientes

**Descripción:** El sistema debe permitir la gestión eficiente de citas, dando información de los horarios y días disponibles.

- **Criterios de éxito:**
  <ul style={{ listStyleType: "circle", paddingLeft: "20px" }}>
    <li>Los clientes pueden reservar, modificar o cancelar citas en línea.</li>
    <li>El sistema genera recordatorios automáticos vía email o SMS (opcional).</li>
    <li>El sistema refleja la disponibilidad en tiempo real, evitando reservas duplicadas.</li>
    <li>Sección de administrador donde se visualizan todas las citas pendientes y que ya se atendió durante cada día.</li>
  </ul>

### Épica 3: Gestión de contenido del sitio Web

**Descripción:** El sistema debe permitir agregar o modificar cierto contenido visible en el sitio web como un cms.

- **Criterios de éxito:**
  <ul style={{ listStyleType: "circle", paddingLeft: "20px" }}>
    <li>Facilidad de Uso: Los administradores pueden acceder fácilmente a un panel de control para editar y actualizar 
    las secciones del sitio web sin requerir la intervención de un programador.</li>
    <li>Gestión de Promociones: Los administradores pueden actualizar y modificar las promociones de forma ágil, 
    reflejándose automáticamente en la página principal o landing page.</li>
    <li>Previsualización de Cambios: Antes de publicar las modificaciones, el CMS debe ofrecer una opción de previsualización 
    para que los administradores vean cómo se reflejarán los cambios en el sitio web.</li>
  </ul>
