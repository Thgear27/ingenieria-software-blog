La estimación de historias de usuario es un paso esencial para asegurar una planificación efectiva en el marco de trabajo Scrum. Permite al equipo de desarrollo identificar el esfuerzo necesario para cada funcionalidad, facilitando la organización y gestión del backlog.

### Historias de Usuario

- **Historia 1:** El usuario visitante podrá visualizar una página web atractiva con colores llamativos, imágenes acorde al negocio y en alta definición para captar su atención.
- **Historia 2:** El usuario podrá navegar por la página mediante una barra de navegación clara y accesible para poder encontrar fácilmente la información que busca.
- **Historia 3:** El usuario podrá conocer la ubicación del negocio mediante un mapa de Google Maps integrado con la ubicación de la sede.
- **Historia 4:** El usuario podrá registrarse mediante vinculación de su cuenta de google.

## HERRAMIENTAS

### Métodos de Estimación

Para estimar las historias de usuario, el equipo Scrum utilizará el método **Planning Poker**. Este método facilita la discusión y el consenso sobre el esfuerzo requerido para cada historia, permitiendo asignar puntos de manera colaborativa.

## SALIDAS

### Estimaciones de Historias de Usuario

El equipo de desarrollo estima el esfuerzo de cada historia de usuario en puntos de historia, utilizando Planning Poker para llegar a un consenso. Estas estimaciones permitirán una mejor planificación de los sprints y la entrega continua de valor al cliente.
